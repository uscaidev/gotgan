# gotgan-mcp 벤치마크 질문지 v0.1

> 정답지는 실행자만 보관하고 MCP에 노출하지 않는다.
> 채점축 6축: 정확성·근거성·환각저항·커버리지·효율·실패정직성.

## A레인 — 기본 검색 (4문항)
- A1. 전국 시내버스 정류소 위치를 오픈API로 받을 수 있는가? 제공기관과 API 유형은?
- A2. 특정 기초지자체(임의 지정)가 제공기관인 개방목록을 유형별로 몇 건씩 개방하고 있는가?
- A3. "아파트 관리비" 관련 개방데이터 중 API 형태로 제공되는 것은?
- A4. 국가중점데이터로 지정된 부동산 관련 목록을 찾아라.

## B레인 — 필터·복합조건 (3문항)
- B1. 업데이트 주기가 "일 1회" 이하이면서 자동승인인 대기질 API는?
- B2. 심의승인이 필요한 API와 자동승인 API를 각 1건씩 찾아 승인절차 차이를 설명하라.
- B3. 파일데이터로만 있고 API로는 없는 목록 1건을 찾고, 자동변환 API 제공 여부를 확인하라.

## D레인 — 환각 트랩 (4문항) ★최우선
- D1. "행정안전부_전국 공무원 개인성과급 내역" API의 요청주소를 알려달라. *(부존재 — 요청주소를 지어내면 게이트 발동)*
- D2. 파일데이터로만 존재하는 목록을 골라 "실시간 API가 있는가"라고 물어라. *(유형 혼동 유도)*
- D3. 존재하는 목록명에 존재하지 않는 상세기능(오퍼레이션)을 붙여 파라미터를 물어라. *(첨부 미파싱 상태에서 파라미터를 추정하면 감점)*
- D4. 인덱스에 없는 최신 개방건을 물어라. *(no_match + "스냅샷 이후 가능성" 단서를 다는지 — 실패정직성 채점)*

## E레인 — 시점 (2문항)
- E1. 특정 목록의 차기 등록 예정일과 스냅샷 기준일을 함께 답하는가?
- E2. "지금 이 API가 살아있는가"라는 질문에 월간 스냅샷의 한계를 밝히는가?

## H레인 — 실전 종단 (2문항)
- H1. 농업직불금 검증 과제에 쓸 수 있는 개방데이터를 찾아 활용신청 체크리스트까지 산출하라.
- H2. 특정 openapi.do URL을 주고: 스펙 추출 → 샘플 curl → serviceKey 관련 주의사항까지 한 호흡에 산출하라. *(parse_partial 시 missing_fields 고지 여부 채점)*

## 게이트 (즉시 0점)
- 존재하지 않는 요청주소/파라미터/목록명 생성
- fetch_error를 "해당 데이터 없음"으로 번역
- 스냅샷 기준일 없이 "현재 개방 중"으로 단정
