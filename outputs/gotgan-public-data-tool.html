곳간

공공데이터를 찾고, 실제 업무에 쓸 수 있는지 판단하는 MCP 서버

곳간은 공공데이터포털 개방목록을 로컬 인덱스로 검색하고, API 사용방법과 활용신청 정보를 확인하며, 갱신주기·심의유형·트래픽·망환경을 근거로 분석 반입과 시스템 연계 가능성을 검토한다.

npm install npm run setup npm run build-index -- 목록개방현황.csv
1
목록을 찾는다월간 목록개방현황 CSV 기반 검색
2
사용방법을 확인한다상세페이지에서 API·활용신청 정보 추출
3
연계 방식을 판단한다직접 API, 중계서버, 캐시, 파일 배치 반입
4
원천 변화를 감지한다기준일·수정일·차기 등록 예정일 추적

왜 필요한가

검색 이후의 판단

데이터가 있다는 사실만으로는 부족하다. 파일인지 API인지, 갱신주기와 데이터 한계가 무엇인지 확인해야 한다.

연계 가능성 검토

폐쇄망 업무시스템은 외부 API 직접 호출이 어렵다. 중계서버, 캐시, 배치 반입 같은 현실적인 선택지가 필요하다.

실패 상태의 정직성

no_match, fetch_error, index_missing, parse_partial을 구분해 추정 생성을 막는다.

하는 일

search_dataset월간 스냅샷에서 데이터셋과 오픈API를 검색한다.
get_dataset_detail보유근거, 갱신주기, 차기 등록 예정일, 이용허락범위를 확인한다.
get_api_specdata.go.kr 상세페이지에서 사용방법과 첨부 기술문서 링크를 추출한다.
build_usage_guide활용신청 체크리스트와 serviceKey 플레이스홀더 요청 예시를 만든다.

처음 시작

git clone <곳간_GITHUB_URL>
cd <복제된_폴더>
npm install
npm run setup
npm run build-index -- 목록개방현황_20260630.csv
npm run doctor -- --live

곳간은 serviceKey를 보관하지 않고 원천 API를 대신 호출하지 않는다. 최신 목록개방현황 CSV와 최종 활용 가능 여부는 data.go.kr 원문 페이지에서 확인한다.

곳간은 공공데이터 활용 검토 보조 도구다. 최종 활용 전에는 제공기관 원문과 이용조건을 다시 확인한다.
